# Prompt de Design — Produto White Label (Landing / Showcase)

**Uso:** colar este prompt diretamente no Claude Design  
**Objetivo:** criar uma página de apresentação moderna e comercial da plataforma white-label  
**Público-alvo do design:** o operador/revendedor que quer oferecer o produto para seus clientes

---

## PROMPT PARA O CLAUDE DESIGN

Crie uma **landing page moderna e premium** para uma plataforma SaaS chamada **"FluxoLabel"** — uma solução white-label que permite entregar um sistema de gestão com identidade visual própria para cada cliente, sem duplicar código.

---

### Identidade visual da plataforma

- **Nome:** FluxoLabel
- **Tagline:** "Um sistema. A cara de cada negócio."
- **Cor primária:** `#6C3CE1` (violeta elétrico)
- **Cor de acento:** `#00E5A0` (verde neon suave)
- **Fundo:** `#09090B` (preto profundo, quase carvão)
- **Fundo de cards:** `#111113` com borda `rgba(255,255,255,0.06)`
- **Texto principal:** `#F4F4F5`
- **Texto secundário:** `#A1A1AA`
- **Gradiente de destaque:** `linear-gradient(135deg, #6C3CE1, #00E5A0)`

**Tom visual:** tech premium, confiante, moderno — como se fosse a Vercel ou Linear do segmento de ERP para pequenos negócios. Não parece software de gestão antigo. Parece produto de 2026.

### Estrutura da página (seções em scroll vertical)

#### SEÇÃO 1 — Hero (acima da dobra)

Layout de duas colunas: texto à esquerda, visual interativo à direita.

**Coluna esquerda:**
- Badge pill no topo: `✦ Plataforma White Label` — fundo violeta com borda tênue
- Headline em destaque (fonte bold, tamanho grande):
  ```
  Um sistema.
  A cara de cada
  negócio.
  ```
  A palavra "cara" em gradiente violeta→verde neon
- Subtítulo em cinza claro:
  ```
  Entregue um produto de software com identidade visual própria
  para cada cliente — sem fork, sem branch, sem retrabalho.
  ```
- Dois botões:
  - Primário: "Ver demonstração" — fundo gradiente violeta→verde, texto branco
  - Secundário: "Falar com a equipe" — borda violeta suave, texto cinza claro
- Abaixo dos botões: linha discreta com logos/nomes fictícios de clientes:
  `Bella Corte · Mesa Viva · Boa Praça · Crescer Bem · Movimento Particular`
  com prefixo "Usado por:"

**Coluna direita:**
- Mockup de browser/janela flutuante mostrando três variantes da mesma tela de login simultaneamente (efeito de sobreposição/fan):
  - Versão Bella Corte: fundo preto, logo dourada
  - Versão Boa Praça: fundo branco, vermelho e amarelo
  - Versão Mesa Viva: fundo creme, verde escuro
- Efeito: cards levemente rotacionados em leque, com sombra profunda
- Detalhe: brilho violeta sutil atrás dos cards (glow effect)

Título da seção: "Por que FluxoLabel?"

**Card 1 — Uma base, infinitos clientes**
- Ícone: cubo 3D com ramificações

- Ícone: raio/lightning

### Especificações de UI

- **Fonte:** Plus Jakarta Sans (títulos) + Inter (corpo) — ou similar
- **Tamanhos:** headline hero 56–64px, subtítulo 18px, corpo 15px
- **Border-radius:** 16px em cards grandes, 12px em cards menores, 8px em botões
- **Sombra dos cards:** `box-shadow: 0 0 0 1px rgba(255,255,255,0.05), 0 8px 32px rgba(0,0,0,0.4)`
- **Glow violeta:** `box-shadow: 0 0 40px rgba(108,60,225,0.3)` — usado em elementos hero
- **Espaçamento:** generoso — padding de seção mínimo 120px vertical
- **Animação sugerida:** entrada por scroll (fade + translate-y) em cada seção
- **Modo:** dark mode exclusivo — esta é a identidade da plataforma
- **Largura máxima do conteúdo:** 1200px centralizado
- **Responsividade:** desktop first, mas hero colapsável para mobile

---

### Tom e referências visuais

O visual deve lembrar:
- **Vercel** — dark mode premium, tipografia forte, muita respiração
- **Linear** — interface limpa, cores de acento neon, grid bem construído
- **Stripe** — confiança, clareza, zero poluição visual

Evitar:
- Paletas coloridas demais (mais de 2 cores de acento)
- Cards com muito conteúdo interno
- Visual de "painel de controle ERP antigo"
- Bordas arredondadas exageradas (não é app mobile)
